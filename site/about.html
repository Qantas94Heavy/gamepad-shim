How does it work?

The HTML5 Gamepad Shim uses a combination of vendor prefixes, such as webkit and moz, and where the browser doesn't support the native API at all (IE and Safari), utilises a Flash implementation that connects with the browser to provide an equivalent API. Long story short, it makes things too easy to do.
